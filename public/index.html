Go Templating via Hugo in 3 Minutes. Let's Go!

  1. Go has its own templating language: https://pkg.go.dev/text/template
  2. Go templating is not particularly easy to lear or read. The fundamentals are easy to learn, but beyond the basics it’s a painful.
  3. There are superior JavaScript-based options for creating UI’s (e.g. React, Vue, Handlebars, etc.)
  4. The one context where Go templating shines above the rest is via Hugo - a static site generator written in Go.
  5. Hugo is by far the fastest static-site generator available today. And it’s orders of magnitude faster than more popular React-based options like Next.js and Gatsby.
  6. Hugo setup is dead simple, it comes with a variety of features and configuration for advanced customization.
  7. If you want to avoid all the fancy stuff, there are hundreds of free ready-to-go themes you can drop in and use.
  8. If we're going to build a bunch of Go apps that need documentation. We should consider Go Templating + Hugo + CSS Framework.
  9. The only obstacle is learning Go templating, which is a low barrier to entry.
  10. There CMS options for non-developers create, edit, and maintain content.
